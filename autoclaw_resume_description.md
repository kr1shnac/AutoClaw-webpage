# AutoClaw — Resume Project Description

## Pattern Analysis (Kavya's Style)

Here's what makes Kavya's project descriptions effective:

| Element | Pattern |
|---|---|
| **Title line** | `PROJECT NAME — One-liner describing what it is \| Tech Stack` + optional award/link |
| **Opening bullet** | Starts with a strong action verb ("Developed", "Designed") + describes the *what* and *why* in one confident sentence |
| **Feature bullets** | Each bullet highlights ONE feature, starting with an action verb, with 1-2 **bolded keywords** that catch the recruiter's eye |
| **Impact language** | Every bullet ends by tying the feature back to a tangible outcome (efficiency, engagement, experience) |
| **Bolding strategy** | Only the most recruiter-relevant terms are bolded — technical wins, domain keywords, measurable outcomes |
| **Tone** | Authoritative, concise, no filler — reads like a press release, not a diary |
